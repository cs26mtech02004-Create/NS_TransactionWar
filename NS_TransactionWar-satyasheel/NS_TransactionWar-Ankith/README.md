# SecurePay — Secure PHP Banking Application

A full-stack banking web application built with PHP 8.2, MySQL 8, and Apache. Runs in Docker with HTTPS via a self-signed certificate. Themed after retro Indian ATM interfaces (SBI/PNB era) with a clean phosphor-green terminal aesthetic.

## Database Schema

### users
| Column | Type | Notes |
|--------|------|-------|
| acct_ref | INT UNSIGNED AUTO_INCREMENT | Internal PK — never shown to users |
| acct_token | VARCHAR(12) UNIQUE | 12-char hex, shown to users, used in URLs |
| handle | VARCHAR(30) UNIQUE | Login username — permanent |
| contact_addr | VARCHAR(254) UNIQUE | Email address |
| auth_digest | VARCHAR(255) | bcrypt cost 12 |
| ledger_bal | DECIMAL(12,2) DEFAULT 100.00 | CHECK >= 0 |
| display_name | VARCHAR(100) NULL | Optional real name |
| profile_text | TEXT NULL | Optional biography |
| avatar_file | VARCHAR(64) NULL | Filename only, not path |
| enrolled_at | DATETIME | Account creation time |
| modified_at | DATETIME | ON UPDATE CURRENT_TIMESTAMP |

### transactions
| Column | Type | Notes |
|--------|------|-------|
| txn_ref | INT UNSIGNED AUTO_INCREMENT | |
| debit_acct | INT UNSIGNED FK→users | RESTRICT on delete |
| credit_acct | INT UNSIGNED FK→users | RESTRICT on delete |
| txn_amount | DECIMAL(12,2) | CHECK > 0 |
| txn_note | VARCHAR(500) NULL | Optional comment |
| txn_at | DATETIME | |

Constraints: `debit_acct != credit_acct`

### login_attempts
Tracks failed logins per IP. Rate limit: 5 failures per IP per 30 minutes. Column: `origin_ip`, `attempt_at`.

### activity_log
One row per page visit. Columns: `actor_handle`, `endpoint`, `origin_ip`, `log_at`.

### admins
Separate table from users. Columns: `admin_handle`, `auth_digest`. Admin credentials never share the users table.

## Security Architecture

### HTTPS / TLS

| Item | Detail |
|------|--------|
| Certificate | Self-signed, generated locally with openssl |
| Protocols | TLS 1.2 and 1.3 only — SSLv3, TLS 1.0, TLS 1.1 disabled |
| HTTP port 80 | Permanent redirect to HTTPS — no content served over plain HTTP |
| HSTS | `Strict-Transport-Security: max-age=31536000` on all HTTPS responses |
| Cert storage | Baked into Docker image via `COPY certs/` at build time |
| Cookie secure flag | `session.cookie_secure = true` — cookie never sent over HTTP |

### Authentication

| Threat | Defence |
|--------|---------|
| Brute force | 5 attempts per IP per 30 min, then lockout |
| Username enumeration via timing | Dummy hash compared when user not found |
| Username enumeration via message | Single generic "Invalid credentials" message |
| Session fixation | `session_regenerate_id(true)` on login |
| Session hijacking | httponly + samesite=Strict + secure cookie flags |
| Session survives browser close | Cookie lifetime = 0 (session cookie, not persistent) |
| Idle session never expires | Server-side `_last_active` check in auth_guard — 30 min timeout |
| CSRF on logout | Logout is POST-only with CSRF token |

### Passwords

- `password_hash($pw, PASSWORD_BCRYPT, ['cost' => 12])`
- Minimum 8 chars, maximum 72 (bcrypt truncation limit), requires uppercase + number
- Visibility toggle is client-side only — server never sees the toggle state

### CSRF Protection

- `bin2hex(random_bytes(32))` token per session
- `hash_equals()` comparison (timing-safe)
- Single-use: regenerated after each validation
- All POST forms include `<?= csrf_input() ?>`
- GET requests (search) are read-only and excluded

### User IDs and Column Obfuscation

Internal `acct_ref` (INT) is used only for database foreign keys and JOINs — stays on the server. `acct_token` (12-char hex) is used in all URLs. Transfers use `handle` (username) — no numeric ID ever enters the browser.

All column names are deliberately non-obvious (`auth_digest`, `ledger_bal`, `origin_ip`, etc.) so that partial schema leaks via error messages give an attacker minimal information.

### Money Transfers

1. Sender identity from `$_SESSION['user_id']` — never from form input
2. Recipient resolved by handle server-side
3. `BEGIN TRANSACTION` wraps all three operations
4. `SELECT ... FOR UPDATE` locks sender row (prevents race/double-spend)
5. `UPDATE ... WHERE ledger_bal >= txn_amount` — atomic insufficient-funds check
6. `ROLLBACK` on any failure
7. Transfer to self explicitly rejected

### Image Upload

All checks in `includes/upload_handler.php`:

1. `$_FILES['error'] === UPLOAD_ERR_OK`
2. `filesize($tmp_name)` ≤ 2MB (not `$_FILES['size']` which can be forged)
3. `finfo(FILEINFO_MIME_TYPE)` on actual file bytes — not the filename
4. Extension whitelisted against MIME type
5. `getimagesize()` validates it is a real image
6. Random filename: `bin2hex(random_bytes(16)) . '.' . $ext`
7. Stored in `/var/www/private/uploads/` — outside Apache webroot
8. `chmod($path, 0640)` after move
9. Old image deleted on new upload

### Session Configuration

```
session.cookie_httponly = 1       JS cannot read the session cookie
session.cookie_samesite = Strict  Cookie not sent on cross-origin requests
session.cookie_secure   = 1       Cookie only sent over HTTPS
session.cookie_lifetime = 0       Session cookie — browser close destroys it
session.use_strict_mode = 1       Server rejects unknown session IDs
session.gc_maxlifetime  = 900     30-minute server-side GC
```

Idle timeout: `auth_guard.php` checks `$_SESSION['_last_active']` on every protected page. If `time() - _last_active > 900`, session is destroyed and user is redirected to login with an expiry message.

### Apache Hardening

- Port 80 (`vhost.conf`): `Redirect permanent / https://localhost/` — nothing else
- Port 443 (`ssl-vhost.conf`): full application vhost with TLS, directory blocks, security headers
- `Options -Indexes -FollowSymLinks` everywhere
- `AllowOverride None` + `Require all denied` on `config/` and `includes/`
- `DirectoryIndex login.php` — visiting `/` goes to login

### PHP Hardening (`docker/security.ini`)

```
expose_php         = Off
display_errors     = Off
allow_url_fopen    = Off
allow_url_include  = Off
open_basedir       = /var/www/html:/var/www/private:/tmp
disable_functions  = exec,passthru,shell_exec,system,proc_open,...
```

### Container Security

```yaml
security_opt:
  - no-new-privileges:true
cap_drop:
  - ALL
cap_add:
  - NET_BIND_SERVICE
```

MySQL has no `ports:` mapping — unreachable from outside Docker network.

## Running the Application

### 1. Create the env file

```bash
cp .env.example .env
# Edit .env — set DB_PASS and MYSQL_ROOT_PASSWORD
# Generate strong values with: openssl rand -base64 32
```

### 2. Generate the TLS certificate

Run this once inside your project folder before building:

```bash
mkdir -p certs && openssl req -x509 -nodes -days 365 -newkey rsa:2048 \
  -keyout certs/key.pem \
  -out    certs/cert.pem \
  -subj "/C=IN/ST=Telangana/L=Hyderabad/O=SecurePay/CN=localhost"
```

Verify the files exist before the next step:

```bash
ls -lh certs/
```

Both `cert.pem` and `key.pem` must be present and non-zero. The Dockerfile `COPY`s them into the image at build time — if they are missing, the build will fail.

### 3. Build and start

```bash
docker compose up --build -d
```

The `--build` flag is required on first run and any time you regenerate the certificate. Without it Docker reuses the cached image and the cert files will not be updated inside the container.

### 4. Accept the browser warning

Visit `https://localhost`. You will see a security warning because the certificate is self-signed rather than issued by a trusted CA. This is expected. The connection is still fully encrypted.

- Chrome: click **Advanced** → **Proceed to localhost (unsafe)**
- Firefox: click **Advanced** → **Accept the Risk and Continue**

You only need to do this once per browser.

### 5. Create admin account

```bash
# Generate bcrypt hash
docker exec securepay_app php -r \
  "echo password_hash('YourAdminPass123!', PASSWORD_BCRYPT, ['cost'=>12]);"

# Open MySQL shell
docker exec -it securepay_db mysql -u root -p

# Inside MySQL:
USE securepay;
INSERT INTO admins (admin_handle, auth_digest) VALUES ('admin', '$2y$12$YOUR_HASH');
EXIT;
```

### 6. URLs

All URLs are HTTPS. HTTP redirects automatically.

| URL | Page |
|-----|------|
| https://localhost/ | Auto-redirect to login or dashboard |
| https://localhost/register.php | Create account |
| https://localhost/login.php | Sign in |
| https://localhost/dashboard.php | Balance + recent transactions |
| https://localhost/profile.php | Edit profile, upload avatar |
| https://localhost/search.php | Find users by username |
| https://localhost/transfer.php | Send money by username |
| https://localhost/transaction_history.php | Paginated full history |
| https://localhost/admin_login.php | Admin sign in |
| https://localhost/admin_logs.php | Activity log viewer |

### 7. Stop

```bash
docker compose down        # keeps database
docker compose down -v     # wipes database and volumes
```

## File Reference

### docker/vhost.conf
Port 80 only. Single directive: `Redirect permanent / https://localhost/`. No content is ever served over plain HTTP.

### docker/ssl-vhost.conf
Port 443. Sets `SSLEngine on`, points to cert and key at `/etc/apache2/ssl/`. Enforces TLS 1.2+ only. Contains all directory blocks (`config/` and `includes/` denied), security headers, HSTS, and cache-control rules.

### certs/
Generated by openssl on the developer's machine. Copied into the Docker image at build time via `COPY certs/` in the Dockerfile. Listed in `.gitignore` — never committed to version control.

### config/db.php
PDO connection. `ATTR_EMULATE_PREPARES => false` forces true prepared statements. `ERRMODE_EXCEPTION` so errors throw rather than silently fail. Credentials from `getenv()` only.

### config/session.php
Sets all session security flags before `session_start()`. Cookie lifetime is `0` — makes it a session cookie that the browser deletes on close. Defines `SESSION_IDLE_TIMEOUT = 900`. Provides `regenerate_session()` and `destroy_session()`.

### includes/auth_guard.php
Protects pages from unauthenticated access. Also enforces 30-minute idle timeout: checks `$_SESSION['_last_active']` on every request, destroys session and redirects if idle too long, otherwise updates the timestamp.

### includes/csrf.php
`generate_csrf_token()` — creates token, stores in session. `csrf_input()` — outputs hidden input field. `verify_csrf_token()` — validates with `hash_equals()`, dies on failure. Token is single-use.

### includes/header.php
Sends security headers, logs the page visit, renders nav bar. Logout is a `<form method="POST">` button — not an anchor tag.

### includes/transfer_handler.php
Accepts `handle` (username), resolves to internal `acct_ref` server-side. Wraps deduct + credit + log in a DB transaction with `FOR UPDATE` row locking.

### includes/upload_handler.php
Five-layer validation. Stores files in `/var/www/private/uploads/` which Apache cannot serve. Returns `['success', 'filename', 'error']`.

### includes/logger.php
Called automatically from `header.php` on every page load. Sanitises log values. Rotates at 10MB. Writes to both flat file and `activity_log` DB table.

### register.php
Generates `acct_token` with `bin2hex(random_bytes(6))` — collision-checked before insert. Sets `ledger_bal` to `100.00` in INSERT — never from user input.

### login.php
Rate-limits by `REMOTE_ADDR`. Compares dummy hash when user not found (timing attack prevention). Sets `$_SESSION['_last_active']` on successful login. Shows session expiry message if redirected by idle timeout.

### logout.php
POST-only. Validates CSRF token. Calls `destroy_session()`. GET requests are silently redirected — logout via URL bar does nothing.

### admin_login.php
Separate `admins` table lookup. Rate limit: 3 attempts per 30 minutes. Sets `$_SESSION['is_admin'] = true` on success — completely separate from user session.

### admin_logs.php
Checks `$_SESSION['is_admin']` — a regular logged-in user cannot access this. Dynamic WHERE with bound parameters. Raw log tail via `SplFileObject`.

## Attack Surface Summary

| Category | Attack | Mitigation |
|---|---|---|
| Transport | Plain HTTP interception | Port 80 redirects to HTTPS permanently |
| Transport | Downgrade attack | HSTS header forces HTTPS for 1 year |
| Transport | Weak TLS | SSLv3, TLS 1.0, TLS 1.1 disabled — TLS 1.2+ only |
| Transport | Cookie sent over HTTP | `cookie_secure = true` — HTTPS only |
| Injection | SQL injection | PDO prepared statements, `EMULATE_PREPARES false` |
| Injection | LIKE wildcard injection | `addcslashes($term, '%_')` |
| Injection | Log injection | Strips `\n \r \v \f \0 \|` from log values |
| Injection | Path traversal in uploads | Random filename, `basename()` on all paths |
| XSS | Stored/reflected XSS | `htmlspecialchars()` on all output, CSP header |
| XSS | Cookie theft via XSS | `httponly` session cookie flag |
| CSRF | Forged POST requests | CSRF tokens on all forms including logout |
| CSRF | Forged logout | Logout is POST-only — GET does nothing |
| Auth | Brute force | 5 attempts/IP/30 min rate limit in DB |
| Auth | Username enumeration (timing) | Dummy hash compared on unknown user |
| Auth | Username enumeration (message) | Generic error message for all failures |
| Auth | Session fixation | `session_regenerate_id(true)` on login |
| Auth | Session hijacking | httponly + samesite=Strict + secure cookies |
| Auth | Session survives browser close | Cookie lifetime = 0 (session cookie) |
| Auth | Idle session never expires | Server-side `_last_active` check, 30 min |
| Auth | Stale session after logout | Three-step destroy: array + file + cookie |
| Auth | Cached page after logout | `Cache-Control: no-store` on all PHP |
| IDOR | Access other users' data | All WHERE clauses use `$_SESSION['user_id']` |
| IDOR | Enumerate accounts by ID | Sequential INT replaced with random `acct_token` |
| Schema | Column name inference | Obfuscated names: `auth_digest`, `ledger_bal`, etc. |
| Transfer | Sender spoofing | Sender from `$_SESSION` — never from POST body |
| Transfer | Race / double-spend | `SELECT FOR UPDATE` + DB transaction |
| Upload | Malicious file execution | Files outside webroot, `open_basedir`, `disable_functions` |
| Upload | MIME type spoofing | Magic byte check via `finfo` |
| Upload | Size forgery | `filesize($tmp_name)` — not `$_FILES['size']` |
| Container | Privilege escalation | `no-new-privileges:true`, `cap_drop: ALL` |
| Container | Direct DB access | No `ports:` on MySQL — internal network only |
| Headers | Clickjacking | `X-Frame-Options: DENY` |
| Headers | MIME sniffing | `X-Content-Type-Options: nosniff` |
| Headers | Server fingerprinting | `expose_php Off`, `display_errors Off` |
