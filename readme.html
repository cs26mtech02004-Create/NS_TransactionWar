Overview

SecurePay Banking Terminal

A full-stack banking web application with defence-in-depth security. PHP 8.2, MySQL 8, Apache 2.4, Docker. Runs on HTTPS using a self-signed certificate generated with openssl. Themed after Indian ATM interfaces of the early 2000s.

Implemented Features
  • HTTPS via self-signed TLS certificate (openssl) вЂ” HTTP port 80 redirects to HTTPS
  • HSTS header вЂ” browser enforces HTTPS for 1 year after first visit
  • Session cookie lifetime = 0 вЂ” browser close destroys session
  • 30-minute idle timeout enforced server-side in auth_guard.php
  • Obfuscated database column names across all tables
  • User registration with bcrypt (cost 12) password hashing
  • Login with per-IP rate limiting and timing-safe comparison
  • Secure session management вЂ” httponly, samesite=Strict, secure, regeneration on login
  • CSRF tokens on every state-changing form including logout
  • Profile editing вЂ” name, bio, email, avatar
  • Secure image upload вЂ” MIME validation, outside webroot, random filename
  • User search by username вЂ” includes own account, no ID exposure
  • Money transfers by username вЂ” internal IDs never touch the browser
  • Atomic transfers with row-level locking вЂ” no race conditions
  • Transaction history with pagination
  • Activity logging to file + database on every page load
  • Admin panel вЂ” log viewer with filtering
  • Docker deployment with dropped capabilities and no exposed DB port
  • Random non-sequential public user tokens (12-char hex)
  • Password visibility toggle on all password fields
  • Custom confirm modal вЂ” no browser alert() or confirm()
  • POST-based logout with CSRF token validation

HTTPS Setup вЂ” Local Development

How It Works

Apache runs two virtual hosts. Port 80 does nothing except redirect every request permanently to HTTPS. Port 443 is the real application vhost with TLS enabled. The certificate is a self-signed cert generated locally with openssl and copied into the Docker image at build time.

Step 1 вЂ” Generate the certificate

Run this once inside your project folder before building. It creates both files in certs/:

mkdir -p certs && openssl req -x509 -nodes -days 365 -newkey rsa:2048 \
  -keyout certs/key.pem \
  -out    certs/cert.pem \
  -subj "/C=IN/ST=Telangana/L=Hyderabad/O=SecurePay/CN=localhost"

Verify the files exist before building:

ls -lh certs/

Both cert.pem (~1.2KB) and key.pem (~1.7KB) must be present. The Dockerfile COPY certs/ line bakes them into the image вЂ” if they are missing, the build fails.

Step 2 вЂ” Build and start

docker compose up --build -d

The --build flag is required on first run and any time you regenerate the certificate. Without it Docker reuses the cached image and the new cert files will not be present inside the container.

Step 3 вЂ” Accept the browser warning

Visit https://localhost. You will see a security warning because the cert is self-signed rather than issued by a trusted CA. The connection is still fully encrypted. Accept once:

BrowserSteps
ChromeClick Advanced в†’ Proceed to localhost (unsafe)
FirefoxClick Advanced в†’ Accept the Risk and Continue
